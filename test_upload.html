Prueba de subida – identificar error 403

Base URL:

1. GET a /api/test_upload.php

Comprueba que las peticiones llegan a PHP.

(sin ejecutar)

2. POST sin archivo

Comprueba que nginx/Apache permiten POST.

(sin ejecutar)

3. POST con archivo (multipart)

Si aquí da 403, el bloqueo es por subida de archivo (nginx/Apache). Si da 200, el problema puede ser solo en adjuntos.php (sesión, ruta, etc.).

(sin ejecutar)

Resumen

  • GET 200 → La ruta /api/ responde.
  • POST sin archivo 200 → POST permitido.
  • POST con archivo 403 → Nginx o Apache bloquean la subida (client_max_body_size, proxy_request_buffering, ModSecurity, etc.).
  • POST con archivo 200 → El servidor permite subir; si adjuntos.php sigue con 403, revisar sesión o otra regla solo para esa URL.
